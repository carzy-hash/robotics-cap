Control interface

Action Chunk 可以被理解成一种 API 很窄的 Code Agent 输出。

它不是错误路线，而是表达能力有限。CapX 路线的目标是把 Action Chunk 从最终答案， 变成策略代码在约束下生成的短动作产物。

Core distinction

差别在哪里

  1. 直接输出动作时，模型的理由大多藏在轨迹里。
  2. 输出策略代码时，目标、约束、检查和恢复逻辑可以先被审计。
  3. Action Chunk 仍然存在，只是变成代码执行后的短动作接口。
