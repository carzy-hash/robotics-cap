Code as Runtime · Context note

Code as Runtime 的主 Agent，
每轮获得什么上下文？

这篇文章定义 Code as Runtime 的 wake-up context：主 Agent 每轮获得哪些输入、这些输入按什么顺序排列、哪些内容跨轮保持稳定，以及 latest image 为什么位于最后。目标是在保留任务执行所需状态的同时，尽量复用多模态推理缓存。

01 · The exact wake-up context

上下文从稳定前缀开始，
沿变化速度向后展开。

这里假设推理服务对多模态输入使用严格的 prefix cache：只有从开头开始完全一致的 token 与图像内容才能复用。任何过早出现的时间戳、queue 余量或新图像，都会让它之后的内容失去缓存机会；目标服务是否缓存 image token，仍需用真实 cache hit 验证。

02 · Where the cache breaks

缓存只复用一条
精确的公共前缀。

Turn N+1 能复用多少，取决于它与 Turn N 从开头起保持一致到哪里。最早发生变化的位置决定了本轮 cache miss 的起点。

03 · Sticky exponential history

指数衰减负责覆盖时间，
粘性槽位负责保住缓存。

历史采样维护一组持久槽位：新帧只进入最近端；较老槽位在跨过时间桶边界或出现重要事件时替换。这样可以同时覆盖长时间跨度并保持公共前缀。

01

旧到新排列相同的旧图像形成最长公共前缀；最近槽位的轮换只影响 suffix。

02

槽位按事件更新时间桶跨界、anchor 触发或预算压缩时，才替换对应 image id。

03

图像与说明一起冻结timestamp、camera id、caption 与 image 必须作为一个不可变单元，否则文字变化同样会击穿 cache。

04

按信息增益淘汰先删除与邻近帧重复的普通图像，保留动作转折、接触和失败证据。

06 · Context builder contract

真正需要实现的，
是确定性的 context builder。

缓存收益最终取决于序列化是否稳定。相同状态必须生成相同前缀；预算不足时也要使用明确、可回放的淘汰顺序。

当前时间归入 current turn。时间戳、request id 和动态统计统一排列在 system / helper 区之后。

所有集合稳定排序。primitive、handler、对象 belief 和 tool result 使用确定 key；禁止依赖 map 的偶然遍历顺序。

history 使用不可变 image unit。image id、caption、camera 与时间一旦进入槽位就一起冻结，直到该槽位被明确替换。

超预算时按价值淘汰。先压缩重复 stdout 与普通帧，再缩短 decision trace；固定保留 safety contract、active code、queue state 和 latest image。

记录 cache lineage。每轮 trace 保存 prefix version、首个变化 block、实际 token 占比与 cache hit，才能验证这套设计是否真的节省推理。

def build_agent_context(snapshot, episode, budget):
    blocks = [
        SYSTEM_POLICY_V3,
        PRIMITIVES_AND_HELPERS_V8,       # immutable, versioned
        episode.frozen_contract(),       # stable inside this episode
    ]

    blocks += episode.history.select_sticky(
        buckets=[64, 32, 16, 8, 4, 2, 1],
        keep_event_anchors=True,
        order="oldest_first",
        budget=budget.history,
    )

    blocks += [
        stable_serialize(snapshot.runtime),
        summarize_recent_delta(snapshot, turns=4),
        current_sensor_summary(snapshot),
        DECISION_REQUEST,
        snapshot.latest_image,          # always the last content block
    ]

    assert blocks[-1].type == "image"
    return blocks
