# Code as Runtime: Brief

## Goal

提出一条可讨论、可证伪、可逐步实现的机器人数据与训练路线：让 Code as Runtime 系统在真机上以推理时搜索、验证与恢复的方式完成任务，把高价值执行轨迹整理为可审计的数据，再用于 SFT、偏好学习、失败建模和后续策略改进。

当前目标不是宣称一套已经被大规模实验验证的方法，而是把问题、系统边界、数据对象和关键假设说清楚，为后续原型和实验协议提供共同语言。

## Working title

中文：**Code as Runtime：从真机推理轨迹到训练数据**

英文：**Code as Runtime: From Real-Robot Reasoning Traces to Training Data**

## Core thesis

真机数据不必只被视为 `(observation, action)` 序列。对于由高层 agent、技能/动作策略、验证器和安全控制器构成的 hybrid 系统，一次执行可以同时产生任务分解、工具调用、短时动作、状态检查、失败原因、恢复决策和最终结果。经过严格的数据清洗、因果边界处理和质量筛选后，这些轨迹可以成为 SFT 及其他训练目标的数据来源。

这条路线的核心价值假设是：**推理时计算和真机交互可以先换来较少但结构更丰富的成功/失败轨迹，再把它们蒸馏为更便宜、更稳定的策略能力。**

## Intended reader

- 机器人学习、VLA、具身智能方向的研究者与工程师。
- 熟悉 SFT / RL / imitation learning，但不一定认同“完整思维链必须进入训练数据”的读者。
- 希望判断这条路线是否值得做小规模原型的研究负责人。

## Publication form

当前阶段使用 **Research Note / Technical Preview（研究札记 / 技术预览）**，而不是完整 preprint。

- 页面保留摘要、系统图、方法模块、假设、风险和实验计划，具有学术项目页的可引用结构。
- 明确标注 `Concept · v0.1 · Not peer reviewed`。
- 不使用虚构的结果表、SOTA 比较或“证明有效”的结论。
- 当最小原型和消融实验完成后，可升级为 arXiv-style preprint；当前网页可继续作为项目页和版本入口。

## In scope

- Code as Runtime 系统的分层定义与执行闭环。
- 真机推理/执行轨迹的数据 schema。
- 从在线轨迹到 SFT 样本的选择、裁剪、去污染和质量控制。
- SFT 之外可利用的数据对象：偏好对、失败分类、过程奖励、verifier、world/action model 数据。
- 训练—部署分布偏移、安全、隐私、成本和可复现性问题。
- 一套最小实验计划与可证伪假设。

## Out of scope

- 宣称已经完成大规模真机 RL 或获得通用机器人能力。
- 把隐藏 chain-of-thought 当作必须记录或公开的训练目标。
- 用 SFT 取代低层控制、系统辨识、安全控制器或动作策略学习。
- 在没有实验证据时比较具体模型的优劣。

## Terminology guardrails

- **Inference RL**：正文首次出现时必须给出工作定义。建议描述为“在推理/执行阶段通过候选生成、环境反馈、验证和选择形成可学习轨迹”，避免暗示标准定义已经统一。
- **Reasoning trace**：优先指可观测的结构化决策记录（子目标、调用、检查、结果和恢复），不等同于模型隐藏思维链。
- **Code as Runtime**：主 Agent 不只输出一次性动作或程序，而是持续编写动作构建代码与响应 handler，使代码成为跨思考周期存在、可观察并可修改的机器人运行时。

## Acceptance criteria for v0.1

- 读者能在一张图中理解“任务 → 在线执行 → 轨迹仓库 → 数据构造 → 训练 → 再部署”的闭环。
- 每一项核心主张都标记为已有事实、路线假设或待验证问题。
- 给出一份机器可实现的最小轨迹 schema 和样本转换例子。
- 单列负面结果、失败轨迹和安全约束，而不是只讲成功数据。
- 给出 3–5 个最小实验及其否证条件。
- 网页在桌面和移动端可读，且明确显示其 early-stage 状态。
