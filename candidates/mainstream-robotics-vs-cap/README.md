# 主流机器人操作流水线与 CAP：Candidates

## Shared baseline

候选稿使用相同的红杯任务和事实边界。第一轮包含三种完整对比稿；第二轮根据反馈，先把主流 BC 动作训练讲清楚，再在后半段引出 CAP。

| Candidate | Reader | Narrative |
| --- | --- | --- |
| `candidate-a.md` | 第一次了解机器人学习的人 | 教科书式：依次解释数据、模型、监督、仿真和评测，再逐项对照 CAP。 |
| `candidate-b.md` | 产品、合作伙伴和普通技术读者 | 工厂式：把主流路线比作“看录像学操作”，把 CAP 比作“先写工单再执行”。 |
| `candidate-c.md` | AI 与机器人系统工程师 | 系统式：围绕模型形态、监督端点、仿真闭环和公平实验展开。 |
| `candidate-d.html` | 第一次了解机器人学习的人 | 第二轮逐步教学版：数据记录 → BC 训练 → 部署输出 → CAP。 |
| `candidate-e.html` | 偏视觉阅读的普通读者 | 第二轮图示先行版：先给完整主流流水线地图，再逐图解释。 |
| `candidate-f.html` | 视觉优先的非专业读者 | 第三轮独立信息图版：使用真实机器人操作照片，把一条 BC 样本和动作监督画出来。 |
| `candidate-g.html` | 第一次了解机器人学习的人 | 第四轮单图版：沿用 D/E 的文字结构，只用一张丰富图示建立训练输入、VLA 模型和 Action Chunk 输出的直觉。 |

候选稿仅供本地 review，不属于 Pages 发布内容。
