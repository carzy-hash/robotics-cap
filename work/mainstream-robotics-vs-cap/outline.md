# 主流机器人操作流水线与 CAP：Outline

1. **先看一个任务**：把桌上的红杯放进托盘；区分人看到的任务与机器收到的信号。
2. **主流路线的一句话版本**：示范数据 → 训练策略模型 → 输入图像与指令 → 输出动作 → 用成功率检查。
3. **数据从哪里来**：真实遥操作、跨机构数据集、仿真和部署回流；说明数据不等于互联网视频。
4. **模型学什么**：行为克隆；VLA 把视觉、语言和动作接在一起；离散动作 token 与连续 Action Chunk 是两种常见出口。
5. **监督发生在哪里**：主要监督“下一步动作像不像示范”，也可加入价值、奖励或成功标签；指出中间意图往往不可见。
6. **怎样运行和评测**：仿真用于批量训练、回放、扰动与失败测试；真机看任务成功率、泛化、效率、安全和恢复。
7. **CAP 的一句话版本**：观察 → 生成策略程序 → 检查监督端点 → 调用短动作接口 → 再观察。
8. **CAP 的模型形态**：强 VLM/LLM 作为 AR 策略生成器；感知与连续动作专家保留；审计器和仿真器成为显式模块。
9. **CAP 的数据形态**：除图像、指令、动作外，保留目标、对象、约束、工具调用、检查结果、恢复分支和失败原因。
10. **CAP 的仿真闭环**：同一策略程序在仿真中执行；在每个监督端点注入扰动、判错、重试并生成训练记录。
11. **同表对比**：两条路线在学习目标、监督粒度、失败定位、迁移和代价上的差异。
12. **优势与代价**：CAP 的优势是可审计和数据闭环；代价是延迟、接口工程、标签质量与 sim-to-real。
13. **如何公平验证**：相同任务、数据预算、动作底座和硬件，对比成功率、恢复率、违规率、诊断时间与迁移成本。
